# 📄 Registro de Supuestos y Restricciones

## 🧠 Proyecto:
**Sistema web inteligente para la optimización del proceso de matrícula universitaria mediante un módulo de recomendación de horarios basado en inteligencia artificial de la Universidad Continental**

---

## 📌 Supuestos del proyecto

Los siguientes supuestos se consideran verdaderos para el desarrollo del proyecto:

1. Se asume que los estudiantes cuentan con acceso a internet para utilizar el sistema web.
2. Se asume que los usuarios ingresarán correctamente sus preferencias y disponibilidad horaria.
3. Se asume que la información de cursos, horarios y secciones puede ser simulada de forma realista.
4. Se asume que el sistema será utilizado principalmente durante el periodo de matrícula académica.
5. Se asume que el algoritmo de recomendación podrá generar combinaciones válidas dentro de un tiempo aceptable.

---

## ⛔ Restricciones del proyecto

Las siguientes restricciones limitan el desarrollo del sistema:

1. **Tiempo limitado:** El proyecto será desarrollado en un periodo académico de 16 semanas.
2. **Acceso a datos:** No se contará con acceso a bases de datos reales de la universidad.
3. **Alcance académico:** El sistema tendrá fines demostrativos y no será implementado en producción.
4. **Recursos humanos:** El equipo de desarrollo está conformado por un número reducido de estudiantes.
5. **Recursos tecnológicos:** Se utilizarán herramientas y tecnologías accesibles (preferentemente open source).
